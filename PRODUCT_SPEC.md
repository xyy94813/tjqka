# 产品规格文档：德州扑克 PWA 胜率计算器

## 1. 产品概述

该应用是一款面向德州扑克玩家的 Progressive Web App（PWA），帮助用户快速评估当前手牌在已翻开公共牌情况下的胜率，并展示常见高胜率牌型与所有口袋牌型的 open 前概率参考。

## 2. 核心目标

- 提供德州扑克手牌胜率评估
- 支持基于用户当前手牌和已翻开的公共牌计算胜率
- 展示 open 胜率 > 35% 的高胜率牌型以及所有对子胜率参考
- 作为 PWA 实现，支持离线访问、安装到主屏幕和移动体验

## 3. 目标用户

- 休闲德州扑克爱好者
- 想要提高手牌决策准确性的玩家
- 希望快速查看 open 前口袋牌与高胜率牌型的用户

## 4. 关键功能

### 4.1 手牌与公共牌输入
- 手牌输入采用“先选花色，再输入牌号”的组合方式，便于快速选择。
- 花色选择使用 emoji 表示，提升视觉识别效率。
- 公共牌输入支持 0 到 5 张已翻开的公共牌。
- Flop 三张牌单独一行显示，Turn / River 另排显示。
- 牌号支持文本输入，格式为 `2-9、10、J、Q、K、A`。
- 输入 UI 应避免重复牌，并实时校验非法牌面。
- 提供“重置”按钮，方便用户清空并重新输入。

### 4.2 胜率计算
- 计算当前手牌在给定公共牌情况下的胜率。
- 返回主胜率指标：胜率、平局概率、失败概率。
- 默认计算方式为前端本地计算，采用轻量枚举或 Monte Carlo 模拟。
- 仅针对单手牌与随机对手进行计算，不包含用户自定义对手范围。
- 支持实时更新结果，公共牌或手牌变更即可自动重新计算。

### 4.3 结果展示
- 清晰展示胜率、平局概率和失败概率。
- 显示当前基础策略建议：主动进攻 / 保守观察 / 谨慎弃牌。
- 展示当前牌型概率分布，给出手牌可能达成的牌型比例。
- 左侧展示“open 胜率 > 35% 的高胜率牌型”和“所有对子胜率参考”。
- 页面采用三列布局，左侧固定参考数据、中间是操作输入区、右侧显示计算结果。

### 4.4 PWA 能力
- 支持 Web App Manifest 和“添加到主屏幕”功能。
- 支持离线缓存关键页面、静态资源和计算模块。
- 离线状态下仍可输入牌面并进行本地胜率计算。
- 响应式布局，适配手机、平板与桌面屏幕。

## 5. MVP 范围

### 必须实现
- 手牌与公共牌输入界面，包括牌面选择、输入校验与错误提示。
- 胜率计算引擎，基于当前手牌和已翻牌的本地计算。
- 结果展示与实时更新，包含胜率、平局率和失败率。
- 左侧参考面板，展示 open 胜率 > 35% 的高胜率牌型与所有对子胜率。
- PWA 安装提示与基础离线缓存，离线时仍可输入和计算。

### 可选/后续迭代
- 更精细的 open preflop equity 表格。
- 多玩家局势模拟与对手范围分析。
- 历史记录与牌局复盘功能。
- 更丰富的可视化图表与策略热力图。

## 6. 用户流程

1. 用户打开 PWA。
2. 进入“胜率计算器”页面。
3. 选择或输入两张手牌。
4. 选择或输入已翻开的公共牌。
5. 系统自动计算并展示当前胜率与策略建议。
6. 左侧参考区显示高胜率开局牌型和所有对子胜率。

## 7. 成功指标

- 计算结果准确性：胜率计算符合已知牌局概率模型。
- 交互效率：用户在 3 步内完成输入和查看结果。
- 参考数据有效性：高胜率牌型和对子胜率展示与常见开局数据一致。
- PWA 可安装率：支持“添加到主屏幕”并正常运行。
- 响应速度：胜率结果在 1 秒内返回。

## 8. 关键技术要求

- 前端框架：React。
- PWA：manifest、service worker、离线缓存策略。
- 计算模块：纯前端胜率计算或调用轻量本地库，支持 0~5 张公共牌的实时计算。
- 数据验证：确保输入牌面有效、格式正确、无重复。
- 性能要求：计算结果在 1 秒内返回，避免浏览器卡顿。
- 用户体验：提供可视化输入组件、错误提示和结果解释。

## 9. 设计原则

- 简洁：界面直观、步骤少、输入清晰。
- 准确：胜率计算结果可信。
- 快速：尽量避免长时间等待。
- 便捷：支持移动设备、安装与离线使用。

## 10. 未来扩展方向

- 加入“模拟对手范围”功能。
- 支持不同牌局人数和下注阶段分析。
- 提供博弈策略建议和牌型解释。
- 支持多语言与主题模式。

## 11. 前端工程师问题答复

### 业务范围与边界
- 本产品聚焦于单手牌胜率计算，目标是根据用户当前两张手牌和已揭示公共牌，计算该手牌对随机对手的胜率。
- 当前版本不包含对手手牌范围建模或多玩家局势模拟，这些作为后续扩展。
- 成功指标仅针对胜率计算准确性、交互效率、PWA 安装与离线能力。

### 输入与校验
- 推荐采用扑克牌选择器组件，用户可以先选花色再输入牌号。
- 输入组件应实时校验牌面合法性，并在发现重复牌或非法输入时立即提示错误。
- 输入交互应包含“重置”功能，避免用户操作复杂。

### 胜率计算策略
- 初期采用前端本地计算方案，基于轻量枚举或 Monte Carlo 模拟实现胜率计算。
- 计算应覆盖 0~5 张公共牌，重点保证结果在常见牌局下具有可信度。
- 若前端性能受限，可考虑集成轻量计算库或使用 Web Worker 异步执行计算。

### 结果展示与用户理解
- 结果页面应展示胜率、平局概率、失败概率，以及当前牌型概率分布。
- 左侧参考区应展示 open 胜率 > 35% 的高胜率牌型和口袋牌胜率参考。
- 简单策略建议文本应帮助用户理解当前手牌强度。

### PWA 离线能力
- 重点支持静态资源与计算模块离线缓存，保证无网络时仍可输入牌面并进行本地计算。
- 离线模式下的用户体验应保持可安装、可打开、可使用计算结果。
- 网络恢复后可同步最新版本与缓存策略更新。
